1. Introduction

The OLAC Metadata standard [OLAC-Metadata] follows the generic resource description standard known as "Qualified Dublin Core" [DCQ], and supports a powerful extension mechanism described in [OLAC-Extensions]. Any party may develop their own metadata extension and use it in OLAC metadata records. This document lists the third-party extensions which are known to work with OLAC Metadata. Each extension is described in terms of a title and the following five attributes:

Name

The symbolic name that is used as the value of the xsi:type attribute to indicate that the extension is being used in a metadata element.

Date

The latest revision date of the extension or its controlled vocabulary.

Applies to

The Dublin Core elements with which the extension may be used.

Description

A summary description of what the extension is used for.

Documentation

A link to a complete document that defines and exemplifies the extension. If the extension involves a controlled vocabulary, the document should also enumerate and define the terms of the vocabulary.

2. Third party extensions

There are currently no third-party extensions.

3. Hypothetical third party extensions

The following hypothetical third party extensions are provided as examples:

Name:as:language
Date:2002-12-02
Applies to:language, subject
Description: A hypothetical 3rd party extension for the Academia Sinica scheme for identifying Formosan languages. This list is incomplete, and has not been adopted by Academia Sinica.
Documentation:http://www.sinica.edu.tw/formosan/languages.html
Schema:http://www.language-archives.org/devel/third-party/as-formosan.xsd

Codes for categorizing participants

Name:childes:participant
Date:2002-12-02
Applies to:contributor
Description: A hypothetical 3rd party extension for the CHILDES project, to permit age, sex and education of the participant to be represented, from section 4.2 of the CHAT Manual. AGE: This header specifies a speaker's age in years, months, and days, given in the form years;months.days as in 2;11.17 for 2 years, 11 months, and 17 days. To represent a range of several days for a given transcript, use this format: 2;11.17 - 2;11.28. Note that the dash is surrounded by spaces. If you do not know the child's age in days, you can simply use years and months. If you do not know the months, you can use this form: 6; SEX: This header indicates the gender of the speaker, as female or male. EDUCATION: This header specifies the speaker's highest grade level in school. Education is indicated by the integers from 0 to 20, where the numbers after 12 indicate years of college. For example, if the speaker was in the second year of graduate school, this would be represented as 18.
Documentation:http://childes.psy.cmu.edu/manuals/CHAT.pdf
Schema:http://www.language-archives.org/devel/third-party/childes.xsd

Code for Price

Name:netdc:price
Date:2002-12-02
Applies to:rights
Description: A hypothetical 3rd party extension for the "Networking Data Centers" project of the Linguistic Data Consortium and the European Language Resources Association. Price has three attributes, for the decimal value, the currency (USD or EUR) and the membership category (commercial, noncommercial, nonmember).
