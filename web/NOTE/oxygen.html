1. Introduction

<oXygen/> is a full-featured cross-platform XML editor (for Windows, Mac OS X, Linux and Eclipse) providing the tools needed for XML authoring and application development [oXygen]. It is a mature product (now in its Version 9.0 release) that is popular among academics due to its easy-to-obtain academic license that is priced well under $100 [oXygen-Academic]. The fully functional product may be downloaded and freely used for a 30-day evaluation period [oXygen-Download].

This document describes how this product can be used to create and validate a static repository of language resource descriptions that complies with the [OLAC-Repositories] standard. Section 2 describes additional steps that must be done after installation in order to configure <oXygen/> to work with OLAC metadata. Those who create a static repository by running a script over an existing data source do not need a tool like <oXygen/> for creating the repository, but they do need a tool for validating the output of their script. Section 3 describes how <oXygen/> can then be used to validate an OLAC static repository. Finally, section 4 describes how it can also be used to create and maintain a static repository as an original data source.

2. Installation

Download the software for the appropriate platform from [oXygen-Download] and follow the installation instructions provided on that page. An additional step must be taken in order to configure <oXygen/> for use with OLAC metadata. This is to install the XSV (XML Schema Validator) packjage.

The schema validator that is built into <oXygen/> is Xerces, but it is known to fail (incorrectly) with the XML schemas for Qualified Dublin Core [DC-Notes]. Consequently, the default installation of <oXygen/> is not able to validate an OLAC static repository. However, the XSV package does work correctly with the Dublin Core and OLAC schemas and <oXygen/> is pre-configured to work with XSV once it is installed. Download and install the appropriate distribution of XSV:

  • Windows: Download and execute ftp://ftp.cogsci.ed.ac.uk/pub/XSV/XSV210.EXE.

  • Linux: Download ftp://ftp.cogsci.ed.ac.uk/pub/XSV/XSV-2.10-1.noarch.rpm and install with rpm.

The executable path is pre-configured in <oXygen/> for [oXygen-install-dir]/xsv as the installation directory (for instance, in a Windows installation, C:\Program Files\Oxygen XML Editor 9.0\xsv). If it is installed in a different directory the predefined executable path must be corrected in Options / Preferences / Editor / Custom Validation.

It is necessary to close and restart <oXygen/> for the installation of XSV to take effect.

3. Validating a repository

In the <oXygen/> toolbar, following "External Tools", is the Custom Validation button. It says LIBXML by default. Click on the drop-down control to the right of the button and select XSV. The button should now say XSV; clicking the button will invoke XSV schema validation on the current document.

Use File / Open or File / Open URL to open the static repository that you wish to validate. For instance, a sample repository is at the following URL:

http://www.language-archives.org/OLAC/1.0/static-repository.xml

Click on the XSV button to perform schema validation. It is not necessary to load the XML schemas since the schemaLocation attribute in the root element tells the validator where to find all the needed schemas.
