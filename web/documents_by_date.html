OLAC Documents by Date

Latest update: 2006-04-05

This index lists all documents that have come under the OLAC document process. They are listed by latest date of revision, beginning with the most recent. See also OLAC documents by type and OLAC documents by status.

N.B. See the OLAC Process document for an explanation of the document types, status levels, and review process.


2006-04-05   

OLAC Metadata

Standard. This document defines the format used by the Open Language Archives Community OLAC for the interchange of metadata within the framework of the Open Archives Initiative OAI. The metadata set is based on the complete set of Dublin Core metadata terms DCMT, but the format allows for the use of extensions to express community-specific qualifiers.
2006-04-05   

OLAC Process

Standard. This document summarizes the governing ideas of OLAC (i.e. the purpose, vision, and core values) and then describes how OLAC is organized and how it operates.
2003-12-13   

OLAC Language Vocabulary

Proposed Recommendation. This document specifies the controlled vocabulary of language identifiers used by OLAC.
2003-10-10   

OLAC Role Vocabulary

Proposed Recommendation. Role is an attribute of both the Creator and Contributor elements. (Please note that Dublin Core now discourages the use of the Creator element, recommending that all Role information be associated with Contributor elements.) This document specifies the controlled vocabulary of the Role attribute used by OLAC. This attribute specifies the functional role of the entity named in the creation of a given resource.
2003-09-17   

OLAC Repositories

Standard. This document defines the standards OLAC archives must follow in implementing a metadata repository.
2003-07-29   

A query facility for selective harvesting of OLAC metadata

Draft Implementation Note. Documents a verb, Query, supported by the CGI interface to the OLAC Aggregator. The purpose of the verb is to support selective harvesting of OLAC metadata, such as would be needed in order to offer a specialized service based on OLAC metadata. The request returns a ListRecords response; its parameters support the construction of an SQL query to specify the subset of records to harvest.
2003-07-29   

Viser: A virtual service provider for displaying selected OLAC metadata

Draft Implementation Note. Documents a service named Viser, hosted on the OLAC web site, that allows language resource sites to post services based on OLAC metadata without having to implement a conventional service provider. Viser works in conjunction with the query facility of the OLAC Aggregator to selectively harvest OLAC metadata and then provide HTML displays of records that match a query. The service returns an XML document with processing instructions to invoke an XSL stylesheet and set certain display parameters; the actual rendering to HTML happens in the client's browser.
2003-02-15   

Specifications for an OLAC metadata display format and an OLAC-to-OAI_DC crosswalk

Draft Informational Note. Specifies OLAC_Display, the OLAC metadata display format implemented by the OLAC Aggregator service. This format is a reader-friendly view of OLAC metadata that incorporates attribute values into the element content and translates coded values into display labels. The document further specifies the transformation from OLAC_Display format to OAI_DC format.
2003-01-27   

OLAC Discourse Type Vocabulary

Proposed Recommendation. This document specifies the codes, or controlled vocabulary, for the Discourse Type extension of the OLAC Type element. These codes describe the content of a resource as structured in such a way as to represent a particular type of discourse.
2003-01-21   

OLAC Linguistic Subject Vocabulary

Proposed Recommendation. This document specifies the codes, or suggested vocabulary, for the 'linguistic' value of the xsi:type attribute of the OLAC Subject element. These codes describe the content of a resource as about a particular subfield of linguistic science, or about the level of linguistic structure which is the primary concern of a given subfield.
2002-12-12   

OLAC Linguistic Data Type Vocabulary

Proposed Recommendation. This document specifies the codes, or controlled vocabulary, for the Linguistic Data Type extension of the DCMI Type element. These codes describe the content of a resource from the standpoint of recognized structural types of linguistic information.
2002-12-09   

OLAC Protocol for Metadata Harvesting

Former Proposed Standard. This document defines the protocol OLAC service providers use to harvest metadata from OLAC data providers. It defines the responses that OLAC data providers must make to the requests of the protocol.
2002-12-04   

Third Party Extensions

Draft Informational Note. This document lists third-party metadata extensions that are known to be compatible with the OLAC Metadata standards. They have not attained the status of OLAC Recommendations, but data providers are encouraged to try any that seem relevant to the nature of their collection.
2002-11-30   

Recommended metadata extensions

Proposed Recommendation. This document lists all the metadata extensions that are recommended by the Open Language Archives Community for use in describing language resources. (The date issued for this document is the revision date for the text in section 1 and the list of extensions included in section 2. The information about a particular extension may be more recent than this date; see the specific revision date given for each extension.)
2002-11-19   

OLAC Access Vocabulary

Draft Recommendation. Access is an attribute of the Rights element. It provides a broad classification of the ways in which a resource may be used. Users may have to seek more detailed information about a given resource, if Access = restricted, or if they wish to use the resource in a non-standard fashion. Users are always advised to review the specific policies of the data provider before making use any resource. If the Rights element is absent, no assumptions can be made about the status of these and other rights with respect to the resource.
